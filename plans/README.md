## Testing Strategy

Use a layered approach so the cheapest checks run first:

1. `npm run lint` on every plan that touches TypeScript or component logic.
2. `npm run build` after any cross-component or routing change.
3. Small Node-based contract tests for schema, news, AI Elements, and globe
   behavior.
4. Browser smoke with `npm run dev` for overlay and visual-shell work.

- `001` - SQL/table-name contract test, then build smoke.

- `005` - lint + browser smoke for spacing and typography.
