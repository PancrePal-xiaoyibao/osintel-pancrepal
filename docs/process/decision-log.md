- 2026-06-25: Keep the product centered on a Three.js globe interface, `ai-elements`-style assistant components, and Supabase-aligned schema design.
- 2026-06-28: Build the personalized OSINTel as a separate branch and in-app surface (a "My" tab + `/api/personal/*` namespace), leaving the general feed and routes unchanged.
- 2026-06-28: Port the lifescience-research-copilot retrieval and zero-hallucination logic to TypeScript inside the Express server rather than shelling out to the Python `review.py`, to keep the single-Node-process deployment model and reuse existing types and the multi-provider LLM proxy.
- 2026-06-28: Keep the research surface de-identified: only derived gene/cancer tokens are sent to external APIs (Europe PMC, ClinicalTrials.gov); no PatientProfile PII leaves the device.
- 2026-07-02: Adopt an explicit runtime mode contract for production hardening: `real`, `graceful_fallback`, `demo_only`, and `unavailable`. Production-facing APIs must not insert simulated medical content into real feed or chat state when upstream search/model providers are unavailable.
- 2026-07-03: Adopted the production-hardening spec (T1-T7). Backend now uses bcryptjs + signed JWT (15min TTL) instead of plaintext + base64 pseudo-tokens; sensitive routes (manual-ingest, rollback, chat-custom, personal/translate) require auth. Runtime state persisted to SQLite via Node 24's built-in `node:sqlite` (no native compile dependency). LLM gateway has an SSRF guard with a 7-provider host whitelist + private-IP reject. Global error handler returns a consistent envelope `{status,mode,reason,code,requestId}`; structured logging via pino with secret redaction.
- 2026-07-03: Use `node:sqlite` (Node 24 built-in, experimental but stable enough for our single-process model) over `better-sqlite3` to avoid native compilation across Windows/Linux/CI. Note `DatabaseSync` has no `.transaction()` method — wrap in `BEGIN/COMMIT/ROLLBACK` manually.
- 2026-07-03: Default admin/pancreas123 account is dev-only (created when `NODE_ENV !== 'production'` or `ALLOW_DEFAULT_ACCOUNT=1`). Production deploys must set `JWT_SECRET` (>=16 chars) and a strong `DEFAULT_PASSWORD`.
- 2026-07-03: Containerize on node:24-alpine (T5). Node 24 ships `node:sqlite` natively, avoiding the better-sqlite3 native-compile path in Docker. Multi-stage build keeps the runtime image small; non-root user + HEALTHCHECK + graceful SIGTERM (added in T1) make it behave well in compose/k8s.
- 2026-07-03: Caddy over nginx for the reverse proxy (T5). Caddy auto-provisions and renews Let's Encrypt certs with zero config; the Caddyfile is ~30 lines vs. an equivalent nginx+certbot setup of ~100.
- 2026-07-03: Apache-2.0 as the project license (T6). Patent grant matters for medical-adjacent open source; Apache-2.0 is GPL-compatible and the de-facto standard for OSI/FSF-endorsed permissive licensing.
- 2026-07-03: Custom test runner (scripts/run-tests.mjs) over `node --test` (T6). The existing contract tests use side-effect `assert.*` rather than the node:test `test(name, fn)` API, so node --test can't collect them. The runner spawns each file sequentially and aggregates exit codes — cross-platform (no bash dependency).
